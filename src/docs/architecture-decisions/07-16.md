# Architecture Decisions — 16.07.2026

## 1. Spring Boot geçişi PostgreSQL geçişinden önce tamamlanacak

### Karar

İlk aşamada Spring Boot ve REST altyapısı mevcut in-memory repository implementationlarıyla çalıştırılacaktır.

PostgreSQL entegrasyonu daha sonraki bir aşamada yapılacaktır.

### Gerekçe

Spring Boot, HTTP, dependency injection ve PostgreSQL değişikliklerini aynı anda yapmak hata kaynağını belirlemeyi zorlaştırır.

Önce aşağıdaki yapının çalışması doğrulanacaktır:

```text
REST Controller
    ↓
BankService
    ↓
In-memory repository
```

Daha sonra yalnızca outbound repository adapterları PostgreSQL implementationlarıyla değiştirilecektir.

### Sonuç

Spring Boot geçişi ve PostgreSQL entegrasyonu birbirinden ayrı, daha küçük geliştirme adımları olarak yürütülecektir.

---

## 2. CLI korunacak ve REST ayrı bir inbound adapter olarak eklenecek

### Karar

Mevcut `Main` classı şimdilik silinmeyecektir.

CLI ve REST iki ayrı giriş adapterı olarak bulunacaktır:

```text
adapter.in.cli
adapter.in.rest
```

### Gerekçe

CLI hâlihazırda çalışan bir giriş yöntemidir.

Spring Boot’a geçişin amacı CLI kodunu web koduna dönüştürmek değil, BankService’i HTTP üzerinden çağırabilecek yeni bir giriş adapterı oluşturmaktır.

### Sonuç

Uygulama geçici olarak iki farklı giriş noktasına sahip olacaktır:

```text
CLI Main
REST Controller
```

Her iki giriş yöntemi de aynı application service katmanını kullanacaktır.

---

## 3. Spring bean bağlantıları ayrı bir configuration classında tanımlanacak

### Karar

Repository implementationları ve BankService, `BankConfiguration` içindeki `@Bean` methodlarıyla Spring’e tanıtılacaktır.

### Gerekçe

BankService ve repository portlarının doğrudan Spring annotationlarına bağımlı hâle gelmesi istenmemektedir.

Configuration classı sayesinde:

- Domain katmanı Spring’den bağımsız kalır.
- Application service normal Java classı olarak kalır.
- Kullanılacak repository implementationı tek bir yerde seçilir.
- In-memory ve PostgreSQL adapterları daha kolay değiştirilebilir.

### Sonuç

Spring yalnızca nesnelerin oluşturulması ve birbirine bağlanmasından sorumlu olacaktır.

Repository ve BankService methodlarının gerçek davranışları uygulamanın kendi Java kodunda kalacaktır.

---

## 4. REST altyapısı önce basit bir sağlık endpoint’iyle doğrulanacak

### Karar

Gerçek bankacılık endpoint’lerinden önce aşağıdaki basit endpoint oluşturulmuştur:

```text
GET /api/hello
```

### Gerekçe

Bu endpoint, BankService veya repository davranışlarından bağımsız olarak şu parçaların çalıştığını doğrular:

- Spring Boot application başlangıcı
- Gömülü web sunucusu
- Component scanning
- REST controller oluşturulması
- HTTP route eşleştirmesi
- HTTP response üretimi

### Sonuç

Spring Boot altyapısının çalıştığı doğrulandıktan sonra müşteri kaydı, para yatırma ve transfer gibi gerçek use-case endpoint’lerine geçilecektir.
