## 1. Spring Boot Nedir?

Spring Boot, Java ve Spring Framework kullanılarak uygulama geliştirmeyi kolaylaştıran bir altyapıdır.

Özellikle:

- Web uygulaması çalıştırmak
- HTTP isteklerini karşılamak
- Java nesnelerini oluşturmak
- Dependency’leri birbirine bağlamak
- Configuration yönetmek
- Daha sonra veritabanı bağlantısı kurmak

gibi işlemleri kolaylaştırır.

Spring Boot, bankacılık iş kurallarını kendisi yazmaz.

Örneğin aşağıdaki kurallar yine uygulamanın kendi kodunda bulunur:

- Negatif para yatırılamaması
- Yetersiz bakiyeyle para çekilememesi
- Kullanıcının kendisine transfer yapamaması

Spring Boot bu kuralların HTTP üzerinden kullanılabilmesini ve gerekli nesnelerin oluşturulmasını kolaylaştırır.

## 5. BankApplication Classı

Spring Boot uygulamasını başlatmak için şu class oluşturuldu:

```java
package org.example.bank;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class BankApplication {

    public static void main(String[] args) {
        SpringApplication.run(BankApplication.class, args);
    }
}
```

Bu class normal bir Java `main()` methoduna sahiptir.

Spring sistemini başlatan satır:

```java
SpringApplication.run(BankApplication.class, args);
```

Bu işlem sırasında Spring Boot:

- Spring containerını oluşturur.
- Uygulamadaki Spring configurationlarını tarar.
- Bean tanımlarını bulur.
- Gerekli nesneleri oluşturur.
- Dependency’leri birbirine bağlar.
- Web sunucusunu başlatır.
- HTTP isteklerini beklemeye başlar.

---

## 6. `@SpringBootApplication`

```java
@SpringBootApplication
```

annotationı, bu classın ana Spring Boot başlangıç classı olduğunu belirtir.

Bu class kök package altında tutuldu:

```text
org.example.bank
```

Böylece Spring aşağıdaki alt package’ları tarayabilir:

```text
org.example.bank.configuration
org.example.bank.application
org.example.bank.adapter
```

Ana application classının alt package’lardan birinin içine yerleştirilmemesi önemlidir. Aksi durumda bazı Spring bileşenleri tarama alanının dışında kalabilir.

## 7. BankConfiguration Classı

Mevcut repository ve service nesnelerinin Spring tarafından oluşturulması için configuration classı eklendi.

```java
@Configuration
public class BankConfiguration {
}
```

`@Configuration`, Spring’e şu mesajı verir:

> Bu class, Spring tarafından yönetilecek nesnelerin nasıl oluşturulacağını tanımlar.

CLI Main içinde nesneler manuel olarak oluşturuluyordu:

```java
CustomerRepository customerRepository =
        new InMemoryCustomerRepository();

AccountRepository accountRepository =
        new InMemoryAccountRepository();

TransactionRepository transactionRepository =
        new InMemoryTransactionRepository();

BankService bankService = new BankService(
        customerRepository,
        accountRepository,
        transactionRepository
);
```

Spring Boot uygulamasında bu bağlantıların `BankConfiguration` tarafından tanımlanması amaçlandı.

## 8. Bean Kavramı

Spring tarafından oluşturulan ve yönetilen nesnelere bean denir.

Örneğin:

```java
@Bean
public AccountRepository accountRepository() {
    return new InMemoryAccountRepository();
}
```

Bu method Spring’e şunu söyler:

> `AccountRepository` türünde kullanılabilecek bir nesne oluştur ve yönet.

Gerçekte oluşturulan class:

```text
InMemoryAccountRepository
```

olmasına rağmen dışarıya sunduğu sözleşme:

```text
AccountRepository
```

interface’idir.

Bu sayede daha sonra implementation değiştirilebilir:

```text
AccountRepository
    ↓
InMemoryAccountRepository
```

yerine:

```text
AccountRepository
    ↓
PostgreSqlAccountRepository
```

kullanılabilir.

---

## 9. Repository Bean’leri Neden Spring’e Tanıtıldı?

Spring repository methodlarının çalışma şeklini belirlemez.

Örneğin şu kod hâlâ uygulamanın kendi kodudur:

```java
@Override
public Account save(Account account) {
    accounts.put(account.getAccId(), account);
    return account;
}
```

Spring:

- `Map` işlemlerini yazmaz.
- `save()` methodunun içeriğini değiştirmez.
- Repository iş mantığını yönetmez.

Spring yalnızca repository nesnesini oluşturur ve ihtiyaç duyan classa verir.

BankService constructorı:

```java
public BankService(
        CustomerRepository customerRepository,
        AccountRepository accountRepository,
        TransactionRepository transactionRepository
)
```

üç dependency istemektedir.

Spring’in görevi bu interface’lerin hangi implementationlarla karşılanacağını bilmek ve uygun nesneleri constructora vermektir.

## 10. Spring ile Dependency Injection

Daha önce dependency injection manuel olarak yapılıyordu:

```java
BankService bankService = new BankService(
        customerRepository,
        accountRepository,
        transactionRepository
);
```

Spring kullanıldığında aynı işlem Spring container tarafından yapılır.

Akış:

```text
Spring repository bean’lerini oluşturur.
                ↓
BankService bean’ini oluşturur.
                ↓
Repository bean’lerini BankService constructorına verir.
```

Temel dependency injection mantığı değişmemiştir.

Değişen şey, nesneleri oluşturup birbirine bağlayan tarafın artık `Main` yerine Spring olmasıdır.

## 11. Maven Dependency Senkronizasyonu

```java

```

`pom.xml` değiştikten sonra IntelliJ’de:

```text

```

Bu işlem:

- `pom.xml` dosyasını yeniden okur.
- Yeni dependency’leri indirir.
- IntelliJ classpath’ini günceller.
- Spring classlarının tanınmasını sağlar.

`Generate Sources and Update Folders` seçeneği kullanılmadı. Bu seçenek daha çok Maven pluginleriyle otomatik kaynak kod üreten projeler içindir.

## 13. İlk REST Controller

İlk HTTP bağlantısını doğrulamak için `HelloController` oluşturuldu.

```java
package org.example.bank.adapter.in.rest.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api")
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Banka uygulamasi calisiyor.";
    }
}
```

Bu controller henüz BankService kullanmamaktadır.

Amaç yalnızca şu bağlantıyı test etmektir:

```text
HTTP isteği
    ↓
Spring Boot
    ↓
Controller
    ↓
Java methodu
    ↓
HTTP cevabı
```

## 14. Controller Annotationları

### `@RestController`

```java
@RestController
```

Bu classın HTTP isteklerini karşılayan bir REST controller olduğunu belirtir.

Spring bu classı bulur ve nesnesini oluşturur.

### `@RequestMapping`

```java
@RequestMapping("/api")
```

Controller içindeki bütün adreslerin `/api` ile başlayacağını belirtir.

### `@GetMapping`

```java
@GetMapping("/hello")
```

HTTP GET isteğini ilgili Java methoduna bağlar.

Class ve method adresleri birleşir:

```text
/api + /hello = /api/hello
```

## 15. HTTP İsteğinin Çalışma Akışı

Tarayıcıdan şu adres açıldı:

```text
http://localhost:8080/api/hello
```

Tarayıcı uygulamaya şu isteği gönderdi:

```text
GET /api/hello
```

Spring aşağıdaki eşleşmeyi yaptı:

```text
/api
    ↓
HelloController

/hello
    ↓
hello() methodu
```

Spring daha sonra methodu çalıştırdı:

```java
public String hello() {
    return "Banka uygulamasi calisiyor.";
}
```

Dönen String HTTP cevabına çevrildi ve tarayıcıda gösterildi.

Tam akış:

```text
1. BankApplication çalıştırıldı.
2. Spring Boot web sunucusunu başlattı.
3. Sunucu localhost:8080 üzerinde istek bekledi.
4. Tarayıcı GET /api/hello isteği gönderdi.
5. Spring isteğe karşılık gelen controller methodunu buldu.
6. hello() methodu çalıştırıldı.
7. Method bir String döndürdü.
8. Spring bu String’i HTTP cevabı olarak gönderdi.
9. Tarayıcı cevabı ekranda gösterdi.
```

## 16. Localhost ve Port Kavramı

Kullanılan adres:

```text
http://localhost:8080/api/hello
```

Parçaları:

```text
http
→ Kullanılan iletişim protokolü

localhost
→ Uygulamanın kendi bilgisayarımızda çalıştığını belirtir

8080
→ Spring Boot uygulamasının istekleri dinlediği port

/api/hello
→ İsteğin gönderildiği endpoint
```

Uygulama çalıştığı sürece 8080 portunda HTTP isteği bekler.
