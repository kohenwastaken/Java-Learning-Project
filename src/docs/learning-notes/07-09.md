# Learning Notes

## Unit Test, JUnit ve AccountTest

Bugün projede unit test konusuna giriş yaptım. Unit testin ne olduğunu, neden önemli olduğunu, nasıl yazıldığını ve mevcut banking projesinde nasıl uygulanabileceğini öğrendim.

---

### 1. Unit test nedir?

Unit test, kod içindeki küçük bir parçanın tek başına doğru çalışıp çalışmadığını kontrol eden otomatik testtir.

Bu projede test edilebilecek unit örnekleri:

```text
Account.moneyDeposit()
Account.moneyWithdraw()
Account.moneySend()
Account.moneyReceive()
BankService.registerCustomer()
BankService.depositToAccount()
BankService.transferFromAccount()
```

Unit test temel olarak şu soruyu sorar:

```text
Belirli bir input verildiğinde method beklenen sonucu üretiyor mu?
```
Örnek:

1000 TL bakiyesi olan hesaba 500 TL yatırılırsa:
- method başarılı sonuç dönmeli
- bakiye 1500 TL olmalı

---

### 2. Unit test neden önemlidir?

Kod büyüdükçe manuel test yapmak zorlaşır.

Örneğin her değişiklikten sonra elle şu adımları yapmak zaman kaybettirir:

```text
Programı çalıştır
Kayıt ol
Login ol
Para yatır
Para çek
Transfer yap
Bakiye kontrol et
Transaction log kontrol et
Hatalı değerleri dene
```

Unit test sayesinde bu kontroller otomatik yapılabilir.

Unit testin faydaları:

```text
Kodun doğru çalıştığını gösterir.
Refactor sonrası eski davranışın bozulup bozulmadığını gösterir.
Hataları erken yakalatır.
Business logic kurallarını dokümante eder.
Spring Boot / PostgreSQL gibi büyük geçişlerden önce güvenlik ağı oluşturur.
```

### 3. JUnit nedir?

Java’da unit test yazmak için kullanılan en yaygın test frameworklerinden biridir.

Projede JUnit dependency’si Maven üzerinden eklendi.

Test classları şu klasör altında tutulur:

src/test/java

Ana kodlar ise şu klasörde kalır:

src/main/java

### 5. AAA Pattern

Test yazarken AAA pattern kullanıldı.

AAA şu anlama gelir:

```text
Arrange -> Test ortamını hazırla
Act     -> Test edilen methodu çalıştır
Assert  -> Sonucu kontrol et
```

Örnek:

```java
@Test
void moneyDeposit_whenAmountIsPositive_shouldIncreaseBalance() {
// arrange
Account account = new Account(1, BigDecimal.valueOf(1000));

    // act
    boolean result = account.moneyDeposit(BigDecimal.valueOf(500));

    // assert
    assertTrue(result);
    assertBigDecimalEquals(BigDecimal.valueOf(1500), account.getBalance());
}

```
Bu test şunu kontrol eder:

1000 TL bakiyesi olan hesaba 500 TL yatırılırsa:
- method true dönmeli
- bakiye 1500 olmalı

### 6. Test isimlendirme düzeni

Test methodları açıklayıcı olacak şekilde yazıldı.

Kullanılan isimlendirme formatı:

```text
methodName_whenCondition_shouldExpectedResult
```

Örnekler:

```java
moneyDeposit_whenAmountIsPositive_shouldIncreaseBalance
moneyDeposit_whenAmountIsZero_shouldFailAndKeepBalanceSame
moneyWithdraw_whenBalanceIsInsufficient_shouldReturnInsufficientBalanceAndKeepBalanceSame
moneySend_whenAmountIsValid_shouldDecreaseBalance
moneyReceive_whenAmountIsNegative_shouldFailAndKeepBalanceSame
```

Bu isimlendirme sayesinde test methodunun neyi kontrol ettiği adı okununca anlaşılır hale gelir.

### 7. Sadece başarılı senaryolar değil, başarısız senaryolar da test edildi

Bugün sadece başarılı işlemler değil, hatalı durumlar da test edildi.

Test edilen bazı durumlar:

```text
Pozitif para yatırma başarılı olmalı.
0 TL yatırma başarısız olmalı.
Negatif para yatırma başarısız olmalı.
Yeterli bakiyeyle para çekme başarılı olmalı.
0 veya negatif miktarla para çekme başarısız olmalı.
Yetersiz bakiyeyle para çekme başarısız olmalı.
Geçerli miktarla para gönderme bakiyeyi azaltmalı.
0 veya negatif miktarla para gönderme başarısız olmalı.
Yetersiz bakiyeyle para gönderme başarısız olmalı.
Pozitif para alma bakiyeyi artırmalı.
0 veya negatif para alma başarısız olmalı.
```

Bu önemli çünkü gerçek hataların büyük kısmı edge case denilen sınır durumlarda ortaya çıkar.

### 8. Başarısız işlemlerde bakiye değişmemeli

Bugün öğrendiğim önemli noktalardan biri, başarısız işlemlerde sadece result kontrolünün yeterli olmadığıdır.

Örneğin negatif para yatırma işleminde sadece `false` dönmesini kontrol etmek yeterli değildir.

Aynı zamanda bakiyenin değişmediği de kontrol edilmelidir.

Örnek:

```java
@Test
void moneyDeposit_whenAmountIsNegative_shouldFailAndKeepBalanceSame() {
Account account = new Account(1, BigDecimal.valueOf(1000));

    boolean result = account.moneyDeposit(BigDecimal.valueOf(-500));

    assertFalse(result);
    assertBigDecimalEquals(BigDecimal.valueOf(1000), account.getBalance());
}
```

Bu test iki şeyi garanti eder:

```text
Negatif para yatırma reddedilir.
Bakiye eski halinde kalır.
```

### 9. BigDecimal karşılaştırması

Para işlemlerinde `BigDecimal` kullanıldığı için testlerde doğrudan `assertEquals` kullanmak yerine `compareTo` ile karşılaştırma yapıldı.

Sebep:

```text
BigDecimal.equals() scale farkına takılabilir.
BigDecimal.compareTo() matematiksel değeri karşılaştırır.
```

Örneğin:

```text
1500.0
1500.00
```

matematiksel olarak eşittir ama `equals()` açısından farklı değerlendirilebilir.

Bu yüzden test classı içine helper method yazıldı:

```java
private void assertBigDecimalEquals(BigDecimal expected, BigDecimal actual) {
assertEquals(0, expected.compareTo(actual));
}
```

Sonra testlerde şu şekilde kullanıldı:

```java
assertBigDecimalEquals(BigDecimal.valueOf(1500), account.getBalance());

```

Bu kullanım testlerin okunabilirliğini artırdı.

### 10. moneyReceive methodu düzenlendi

Başta `moneyReceive()` methodu `void` idi ve gelen miktarı doğrudan bakiyeye ekliyordu.

Bu durumda negatif değer verilirse bakiye azalabilirdi.

Bu yüzden `moneyReceive()` methodu boolean dönecek şekilde düzenlendi.

Yeni mantık:

```java
boolean moneyReceive(BigDecimal amount) {
if (amount.compareTo(BigDecimal.ZERO) <= 0) return false;

    this.balance = this.balance.add(amount);
    return true;
}
```

Bu sayede:

```text
Pozitif amount -> bakiye artar, true döner.
0 veya negatif amount -> işlem yapılmaz, false döner.
```

Bu değişiklikle `Account` classı kendi kurallarını daha iyi korur hale geldi.

### 12. BankServiceTest başlangıcı

Günün sonunda `BankServiceTest` yazılmaya başlandı fakat tamamlanamadı.

İlk olarak `registerCustomer` testi yazıldı.

Bu testte şunlar kontrol edildi:

```text
Customer null değil mi?
Customer ID doğru mu?
Customer name doğru mu?
Customer surname doğru mu?
Başlangıç bakiyesi 1000 mi?
```

Örnek:

```java
@Test
void registerCustomer_shouldCreateCustomerWithInitialBalance() {
BankService bankService = new BankService();

    Customer customer = bankService.registerCustomer("Arif", "Yılmaz", "123");

    assertNotNull(customer);
    assertEquals(1, customer.getAccId());
    assertEquals("Arif", customer.getName());
    assertEquals("Yılmaz", customer.getSurname());
    assertBigDecimalEquals(BigDecimal.valueOf(1000), bankService.showBalance(customer.getAccId()));
}
```

BankService testlerine sonraki gün devam edilecek.
