# Para Yatırma, Para Çekme ve Transfer Endpoint’leri

## Günün Amacı

Bu çalışmada hesap bakiyesini değiştiren temel bankacılık işlemleri REST API’ye bağlandı:

- Para yatırma
- Para çekme
- Hesaplar arası para transferi

Her işlem için ayrı request DTO’su ve endpoint oluşturuldu.

## HTTP Methodu Seçimi

Bakiye görüntüleme yalnızca veri okuduğu için `GET` kullanır.

Para yatırma, para çekme ve transfer işlemleri sistemin durumunu değiştirdiği için `POST` kullanır.

```text
GET  → veri okuma
POST → yeni işlem oluşturma veya mevcut durumu değiştirme
```

Para yatırma endpoint’inin yanlışlıkla `@GetMapping` ile yazılması `405 Method Not Allowed` hatasına neden olmuştu. Postman’dan POST gönderildiğinde controller tarafında da `@PostMapping` kullanılmalıdır.

## Para Yatırma

Endpoint:

```text
POST /api/accounts/{accountId}/deposits
```

Örnek JSON body:

```json
{
  "amount": 500
}
```

Request DTO:

```java
public record DepositRequest(
        BigDecimal amount
) {
}
```

Controller akışı:

1. Gönderen hesap ID’si URL’den alınır.
2. Miktar JSON body’den alınır.
3. `BankService.depositToAccount(...)` çağrılır.
4. Sonuç kontrol edilir.
5. İşlem başarılıysa güncel bakiye döndürülür.

```java
DepositResult result =
        bankService.depositToAccount(accountId, request.amount());
```

Geçersiz miktar durumunda:

```text
400 Bad Request
```

başarılı işlemde ise `BalanceResponse` ile güncel bakiye döndürülür.

## Para Çekme

Endpoint:

```text
POST /api/accounts/{accountId}/withdrawals
```

Örnek JSON body:

```json
{
  "amount": 200
}
```

Request DTO:

```java
public record WithdrawRequest(
        BigDecimal amount
) {
}
```

Controller içinde `DepositRequest` yerine `WithdrawRequest` kullanılmalıdır. Her endpoint’in kendi amacını açıkça anlatan DTO’sunun bulunması kodun okunabilirliğini artırır.

Olası sonuçlar:

```text
SUCCESS
INVALID_AMOUNT
INSUFFICIENT_BALANCE
```

Geçersiz miktar ve yetersiz bakiye durumlarında şimdilik `400 Bad Request` dönülmektedir.

## `withdraw` ve `withdrawal` Ayrımı

`withdraw` bir fiildir ve method adı için uygundur:

```java
public ResponseEntity<BalanceResponse> withdraw(...)
```

`withdrawal` ise isimdir ve endpoint kaynağı için uygundur:

```text
/withdrawals
```

Kullanılan isimlendirme:

```text
Method       → withdraw
Endpoint     → /withdrawals
Request DTO  → WithdrawRequest
Result enum  → WithdrawResult
```

## Para Transferi

Endpoint:

```text
POST /api/accounts/{accountId}/transfers
```

URL’deki `accountId`, parayı gönderen hesabı temsil eder.

Örnek JSON body:

```json
{
  "targetId": 2,
  "amount": 300
}
```

Request DTO:

```java
public record TransferRequest(
        int targetId,
        BigDecimal amount
) {
}
```

Gönderen hesap ID’si zaten URL’den geldiği için DTO içinde yeniden tutulmaz. DTO’daki ID alıcı hesabı temsil ettiği için alan adı `accountId` değil `targetId` olarak belirlenmiştir.

Controller çağrısı:

```java
TransferResult result = bankService.transferFromAccount(
        accountId,
        request.targetId(),
        request.amount()
);
```

Olası sonuçlar:

```text
SUCCESS
INVALID_AMOUNT
ACCOUNT_NOT_FOUND
INVALID_SELF_ID
INSUFFICIENT_BALANCE
```

Başarılı işlemden sonra gönderen hesabın güncel bakiyesi alınır ve `BalanceResponse` olarak döndürülür.

## Response Modelinin Yeniden Kullanılması

Para yatırma, para çekme ve transfer işlemlerinin tamamında başarı cevabı olarak mevcut `BalanceResponse` kullanıldı:

```java
public record BalanceResponse(
        int accountId,
        BigDecimal balance
) {
}
```

Bu yaklaşım mevcut ihtiyaç için yeterlidir. İşlem başarıyla tamamlandığında istemci, hangi hesabın etkilendiğini ve yeni bakiyeyi doğrudan görebilir.

## Controller’ın Sorumluluğu

Controller:

- URL ve JSON’dan gelen verileri alır
- `BankService` methodunu çağırır
- Sonuca uygun HTTP cevabını üretir

Controller iş kurallarını kendi içinde uygulamaz. Bakiye kontrolü, geçersiz miktar kontrolü ve transfer kuralları `BankService` ile domain modelinde kalır.

## Tamamlanan Hesap İşlemleri

```text
GET  /api/accounts/{accountId}/balance
POST /api/accounts/{accountId}/deposits
POST /api/accounts/{accountId}/withdrawals
POST /api/accounts/{accountId}/transfers
```

Bu endpoint’ler Postman üzerinden başarıyla test edilmiştir.
