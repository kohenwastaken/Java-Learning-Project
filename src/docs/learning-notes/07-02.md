## Learning Notes

### accID Tipi: int mi String mi?

Bugün `accID` değerinin `int` mi yoksa `String` mi olması gerektiğini düşündüm.

Şu an `accID` sistem tarafından otomatik artırılan bir kimlik değeri olarak kullanılıyor:

```java
int ID = 0;
int accID = ID++;
```

Bu yüzden `int` kullanmak daha mantıklı.

### BigDecimal kullanımı

Para işlemleri için `double` veya `float` yerine `BigDecimal` kullanmanın daha doğru oldğuunu öğrendim.
Çünkü para işlemlerinde hassasiyet önemlidir. `double` beklenmeyen küsurat problemlerine yol açabilir.

```java
BigDecimal balance;
BigDecimal amount = new BigDecimal(sc.nextLine());
```
toplama ve çıkarma işlemler `+` veya `-` ile değil, methodlarla yapılır.

```java
balance = balance.add(amount);
balance = balance.subtract(amount);
```

karşılaştırma için ise `<` , `>` yerine `compareTo()` kullanılır.

```java
balance.compareTo(amount) >= 0
```

### Account class oluşturma

Bakiye ve hsap bilgilerini `Customer` içinde tutmak yerine ayrı class oluşturuldu.

```java
public class Account {
    int accID;
    BigDecimal balance;

    public Account(int accID, BigDecimal balance) {
        this.accID = accID;
        this.balance = balance;
    }
}
```

Bu sayede `Customer` kullanıcı bilgilerini, `Account` ise para ve bakiye bilgilerini temsil eder.

### Customer ve Account ilişkisi

Customer ile Account şimdilik ortak `accID` üzerinden ilişkilendiriliyor.

`Customer.accID = Account.accID`

Login işlemi Customer üzerinden yapılır. Giriş başarılı olduktan sonra ilgili Account, `accountList` içinde aranır.

## Referans Mantığı

`findAffiliatedAccount()` ile bulunan Account nesnesi yeni bir kopya değildir, `accountList` içindeki nesnenin referansıdır.

## Account içine davranış ekleme

Para yatırma işlemi Account classı içine method olarak taşınmaya başlandı.

```java
void addDeposit(BigDecimal amount) {
    this.balance = this.balance.add(amount);
}
```

Konsola yazma ve kullanıcıdan veri alma işlemleri Account içinde tutulmalı, Account sadece bakiye mantığını yönetmeli.
