# Spring Boot REST Controller ve DTO Yapısı

## Günün Amacı

Bu çalışmada komut satırı üzerinden kullanılan bankacılık işlemleri Spring Boot REST API üzerinden erişilebilir hâle getirildi. İlk olarak müşteri kaydı ve bakiye görüntüleme işlemleri HTTP endpoint’lerine bağlandı.

## REST API Akışı

Uygulamadaki temel istek akışı şöyledir:

```text
Postman veya başka bir HTTP istemcisi
        ↓
Controller
        ↓
BankService
        ↓
Repository
        ↓
JSON response
```

Postman bu yapının bir parçası veya frontend’i değildir. REST API’ye istek göndermek ve gelen cevapları incelemek için kullanılan bir API istemcisi ve test aracıdır.

## Controller Katmanı

Controller sınıfları dış dünyadan gelen HTTP isteklerini karşılar. Controller’ın görevi iş kurallarını doğrudan uygulamak değil, gelen veriyi uygun biçimde `BankService` katmanına aktarmak ve sonucu HTTP cevabına dönüştürmektir.

### Kullanılan Annotation’lar

#### `@RestController`

Sınıfın HTTP isteklerini karşılayan bir REST controller olduğunu belirtir. Methodlardan dönen Java nesneleri Spring tarafından otomatik olarak JSON’a çevrilir.

#### `@RequestMapping`

Controller içindeki bütün endpoint’ler için ortak adres başlangıcını tanımlar.

```java
@RequestMapping("/api/accounts")
```

Bu sınıftaki endpoint’lerin tamamı `/api/accounts` adresiyle başlar.

#### `@GetMapping`

Veri okumaya yönelik GET isteklerini karşılar.

```java
@GetMapping("/{accountId}/balance")
```

#### `@PostMapping`

Yeni işlem oluşturmak veya sistemde değişiklik yapmak için kullanılan POST isteklerini karşılar.

```java
@PostMapping("/api/customers")
```

#### `@PathVariable`

URL içindeki değişken değeri method parametresine aktarır.

```java
@GetMapping("/{accountId}/balance")
public BalanceResponse showBalance(@PathVariable int accountId)
```

`/api/accounts/5/balance` isteğinde `accountId` değeri `5` olur.

#### `@RequestBody`

İstek gövdesindeki JSON verisini Java nesnesine dönüştürür.

```java
public CustomerResponse register(
        @RequestBody RegisterCustomerRequest request
)
```

## DTO Kullanımı

REST katmanında gelen ve dönen veriler için domain sınıflarını doğrudan kullanmak yerine DTO sınıfları oluşturuldu.

DTO kullanmanın amaçları:

- API’nin dışarıya açtığı alanları kontrol etmek
- Domain modelini REST katmanından ayırmak
- Şifre gibi dışarıya dönmemesi gereken alanları gizlemek
- İstek ve cevap modellerini daha anlaşılır hâle getirmek

DTO’lar Java `record` yapısıyla tanımlandı.

```java
public record RegisterCustomerRequest(
        String name,
        String surname,
        String password
) {
}
```

Record alanlarına klasik getter yerine alan adıyla erişilir:

```java
request.name()
request.surname()
request.password()
```

## Müşteri Kayıt Endpoint’i

Endpoint:

```text
POST /api/customers
```

Örnek JSON body:

```json
{
  "name": "Ahmet",
  "surname": "Yılmaz",
  "password": "1234"
}
```

Controller gelen verileri `BankService.registerCustomer(...)` methoduna gönderir. Oluşturulan müşteri bir `CustomerResponse` nesnesine çevrilir ve `201 Created` cevabı döndürülür.

Örnek response:

```json
{
  "accountId": 1,
  "name": "Ahmet",
  "surname": "Yılmaz"
}
```

Şifre response içinde yer almaz.

## Bakiye Görüntüleme Endpoint’i

Endpoint:

```text
GET /api/accounts/{accountId}/balance
```

Örnek istek:

```text
GET /api/accounts/1/balance
```

Controller, URL’den gelen hesap ID’sini alarak `BankService.showBalance(accountId)` methodunu çağırır.

Dönen bakiye `BalanceResponse` DTO’suna aktarılır:

```java
public record BalanceResponse(
        int accountId,
        BigDecimal balance
) {
}
```

Örnek response:

```json
{
  "accountId": 1,
  "balance": 1000
}
```

## `ResponseEntity`

`ResponseEntity`, response body ile birlikte HTTP durum kodunu da kontrol etmeyi sağlar.

Örneğin müşteri oluşturma işleminde:

```java
return ResponseEntity.status(HttpStatus.CREATED).body(response);
```

Bu cevap:

```text
201 Created
```

durum koduyla birlikte JSON body döndürür.

## Öğrenilen Temel Ayrım

CLI uygulamasında:

```text
Scanner → switch → BankService → ekrana yazdırma
```

REST uygulamasında:

```text
HTTP request → Controller → BankService → JSON response
```

İş kuralları yine `BankService` içinde kalır. Yalnızca uygulamaya giriş yöntemi CLI’dan HTTP’ye dönüşmüştür.
